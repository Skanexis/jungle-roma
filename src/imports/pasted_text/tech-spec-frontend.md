# 🧾 Техническое задание (Frontend)

---

## 1. Общая концепция сайта

* Одностраничный или многостраничный сайт (SPA предпочтительно)
* Основная структура:

  * Главная страница
  * Раздел "Информация"
  * Раздел "Каталог"
  * Раздел "Контакты"
* Быстрая загрузка, плавная навигация между разделами
* Адаптивность (desktop / tablet / mobile)

---

## 2. Главная страница

### Состав:

* Логотип
* Краткое описание / вводный текст
* Навигационный блок с выбором 3 разделов:

  * Информация
  * Каталог
  * Контакты

### Поведение:

* Плавные переходы при выборе раздела
* Анимации появления элементов при загрузке
* Возможен эффект "входа" в сайт (hero-секция с анимацией)

---

## 3. Каталог

### 3.1 Список товаров

* Отображение в виде карточек
* Каждая карточка содержит:

  * Название
  * Медиа (галерея: фото + видео)
  * Цена (вариативная: за 1г, 5г, 10г и т.д.)
  * Кнопка "Связаться с менеджером"

### Функциональность:

* Кнопка ведёт в Telegram (deeplink или web)
* Возможность открытия карточки товара (детальная страница / модалка)

---

### 3.2 Детальная карточка товара

* Расширенная галерея (свайп / переключение)
* Полная информация о товаре
* Все доступные варианты цен
* Кнопка связи с менеджером
* Возможность закрытия (если модалка)

---

## 4. Контакты

* Основные способы связи
* Быстрые действия (кнопки перехода)
* Минимальный, понятный интерфейс

---

## 5. UI/UX направления (без конкретного дизайна)

### 5.1 Общий стиль

* Тематика: "джунгли"
* Не реализм, а стилизованный (иллюстративный / стикерный подход)
* Все элементы интерфейса должны быть согласованы с темой

---

### 5.2 UI-элементы

Проработать в единой стилистике:

* Кнопки
* Инпуты
* Модальные окна
* Карточки
* Навигация
* Галереи

### Требование:

* Визуальная целостность
* Консистентность во всех состояниях (hover, active, disabled)

---

### 5.3 Фон и окружение

* Анимированный фон (лёгкий, не перегружающий)
* Декоративные элементы:

  * Лианы
  * Листья
  * Джунглевая графика

---

### 5.4 Прелоадер

* Кастомный
* Соответствует общей тематике
* Имеет анимацию
* Появляется при первой загрузке и при ленивой подгрузке

---

## 6. Анимации и взаимодействие

### 6.1 Микроанимации

Обязательная проработка:

* Hover-состояния
* Нажатия (click feedback)
* Появление элементов
* Скролл-анимации

---

### 6.2 Переходы

* Между страницами / разделами
* Открытие / закрытие:

  * модалок
  * карточек
  * меню

---

### 6.3 Поведенческие эффекты

* Плавность всех действий
* Отсутствие резких изменений состояния
* Визуальная обратная связь на каждое действие пользователя

---

## 7. UX-детализация

* Фокус на "ощущении живого интерфейса"
* Внимание к мелочам:

  * загрузка контента
  * пустые состояния
  * ошибки
* Минимум "мертвых зон" без реакции

---

## 8. Производительность

* Оптимизация медиа (lazy loading)
* Плавная работа анимаций (без лагов)
* Минимизация веса фронтенда

---

# 📐 10. Адаптивная сетка (Core Layout System)

---

## 10.1 Брейкпоинты

```
xs: 320–479px   (малые телефоны)
sm: 480–767px   (телефоны)
md: 768–1023px  (планшеты)
lg: 1024–1439px (ноутбуки)
xl: 1440–1919px (десктоп)
xxl: 1920+      (большие экраны)
```

---

## 10.2 Контейнеры

```
.container {
  width: 100%;
  margin: 0 auto;
  padding: 0 16px; (xs–sm)
  padding: 0 24px; (md)
  padding: 0 32px; (lg+)
  max-width:
    sm: 100%
    md: 720px
    lg: 960px
    xl: 1200px
    xxl: 1320px
}
```

---

## 10.3 Сетка (Grid)

### База:

```
display: grid;
grid-template-columns: repeat(12, 1fr);
gap:
  xs: 8px
  sm: 12px
  md: 16px
  lg+: 20–24px
```

---

## 10.4 Правило "НЕ РАЗДУВАТЬ"

* Максимальная ширина контента: **1200px**
* Карточки НЕ растягиваются бесконечно
* Всегда есть ограничение:

```
max-width: 100%
```

---

# 📱 11. Mobile-first логика

## Главный принцип:

👉 **Сначала проектируем под 360px, потом масштабируем вверх**

---

## 11.1 Ограничение высоты

КРИТИЧЕСКОЕ правило:

```
viewport height usage:
80–90vh максимум для любого блока
```

---

## 11.2 Запрет длинных экранов

* На mobile:

  * максимум **1–1.5 экрана**
  * остальное — через модалки / переключения

---

## 12.2 ЗАПРЕТ СКРОЛЛА В МОДАЛКЕ

👉 Делается через структуру:

```
.modal {

  flex-direction: column;

.modal-footer   → фикс
```

* Инфо: остаток
* Кнопка: всегда видна

# 🎛️ 15. Кнопки (под твои ассеты)

---

## 15.1 Базовая кнопка (landscape)

```
width: 240px (desktop)
width: 180px (tablet)
width: 140px (mobile)

height:
desktop: 64–72px
tablet: 56px
mobile: 48px
```

---

## 15.2 Square кнопки (иконки)

```
desktop: 96–120px
tablet: 80px
mobile: 64px
```

---

## 15.3 Структура кнопки

```
.button {
  position: relative;
  display: inline-flex;
  align-items: center;
  justify-content: center;
}

.button-bg {
  position: absolute;
  inset: 0;
  background-image: url(...)
  background-size: 100% 100%;
}

.button-content {
  position: relative;
  z-index: 2;
}
```

---

## 15.4 Состояния

```
.button:hover → background-image: hover
.button:active → pressed
.button:disabled → disabled
```

---

## 15.5 ВАЖНО (чтобы не ломались)

* НЕ использовать padding для размера
* Размер задаётся через width/height
* Текст центрируется flex'ом
